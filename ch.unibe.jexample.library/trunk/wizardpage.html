JExample is a small framework to write unit tests that depend on each other. Brought to you by Adrian Kuhn and Lea Hänsenberger.

Dependent tests are not only more fun to write than isolated ones—they also run faster, reduce code duplication and provide better failure localization. How come? Given dependencies A, B and C had been executed successfully, test T can reuse all of their fixtures. This saves both setup costs and code duplication. On the other hand, if one of the dependencies fails, test T and all its dependents are skipped. This helps to pin-point failures to their very cause.

For more information, please refer to the JExample website or follow us on twitter.
